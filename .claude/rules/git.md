# Git rules

## What stays out of git

Never commit:

- `.env`, `.env.*`, any file with secrets
- API keys, RPC keys, private keys
- Personal notes, drafts, scratch files
- Local paths (`C:\Users\prate\...`)
- Private screenshots or videos
- Random exports, CSV dumps, temp debug files
- `outreach/targets-private.md` (named outreach lives off repo)
- `.DS_Store`, `Thumbs.db`, IDE folders

Add to `.gitignore` before the first commit. Audit the gitignore again before adding any new artifact.

## Commit messages

Format:

```
<type>(<scope>): <short summary in present tense, lowercase>

<optional body that explains why, not what>
```

Types: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `perf`, `security`.

Examples that ship:

- `feat(plinth): add dual oracle median check with 50bps tolerance`
- `fix(vigil): close liquidation race via margin_version nonce`
- `docs(prd): record v0.15 honesty pass`
- `security(coffer): cap per adapter per block notional pull`

Examples that do not ship:

- `update stuff`
- `WIP`
- `more changes`
- `final version`

Body explains why and what failure was prevented. Not a changelog rehash of the diff.

## No AI coauthor lines

Never add any of these. Anywhere. Ever.

- `Co-authored-by: Claude`
- `Co-authored-by: Claude Code`
- `Generated with Claude`
- Any reference to an AI assistant in a commit, PR body, or release note

Same rule for AI generated comments in code that say "// generated by Claude" or similar. Strip them before merging.

## Branches

- `main` is always green and deployable
- Feature branches: `feat/<short-name>` or `fix/<short-name>`
- Long lived: `dev` for sprint integration if useful, otherwise feature branches merge to main directly
- Never push to main without CI green and at least one review for non trivial work

## Pull requests

PR title matches the commit format. PR body covers:

- What changed
- Why now
- How it was tested (Kani, proptest, Foundry, Playwright, manual on Sepolia)
- Any risk or known follow up
- Screenshots or video if UI

Link the PR to a tripwire decision if it is a scope cut.

## CI gates

A PR can merge only when:

- `cargo fmt --check` clean
- `cargo clippy --workspace -- -D warnings` clean
- `cargo test` green
- `forge test` green
- `kani --harness=*` green for the listed invariants
- Subgraph builds
- Frontend `pnpm build` and Lighthouse CI ≥ 90 on mobile
- No new secrets detected by `gitleaks`

Never disable a CI gate to land a PR. Fix the failure or pull the change.

## History hygiene

- Squash on merge for feature branches with messy WIP commits
- Rebase before opening a PR if main has moved a lot
- Do not amend or force push after a PR is in review unless requested

## Destructive commands

Stop and ask before running any of:

- `git reset --hard`
- `git push --force` or `--force-with-lease`
- `git clean -fd`
- `git branch -D` on someone else's branch
- `git filter-branch` or `git filter-repo`

If you must run a destructive command, list the alternatives first and wait for an explicit go.

## Sign offs

- Hooks must not be skipped (`--no-verify` is forbidden by default)
- If a hook fails, fix the issue. Do not amend around it.
- If signing is required and fails, fix the key. Do not commit unsigned.

## Tags and releases

- Semver per package (`@atrium/plinth-0.2.0`)
- Tag annotated with release notes:

```
v0.2.0 — plinth dual oracle

Highlights:
- Median(Chainlink, Pyth) with 50bps tolerance
- 5 Kani+proptest invariants in CI

Risks:
- Pyth equity feeds Sepolia gap (mitigated via Praetor relay)

Verification:
- Arbiscan tx 0x...
- Halmos badge: not applicable (Solidity tool, we use Kani)
```

Post deploy summary lands in Discord automatically via CI.

## Public repo readiness

Before flipping a repo public:

- Audit history for any secret, even in old commits (`gitleaks --log-opts="--all"`)
- Strip outreach targets, personal notes, internal probability tables
- Confirm PRD §28.4 sanitization is applied to any PRD shipped in repo
- Add LICENSE, SECURITY.md, CONTRIBUTING.md, CODE_OF_CONDUCT.md

Public history is forever. If a secret was ever committed, rotate it and rewrite history.
